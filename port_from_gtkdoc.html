Porting documentation from GtkDoc

NOTE: This documentation will consider that the project that is being ported uses meson as a build system.

The porting is done in two stages:

  1. We use the Hotdoc smart indexing feature to generate the documentation
  2. run the gi/transition_scripts/hotdoc_gtk_doc_porter script passing all the information from previous GtkDoc setup

Naive Hotdoc based documenation generation

We will use the mesonbuild hotdoc module to generate the documentation.

For the purpose of this porting guide, I will use Gtk as an example. We will start with the gdk library as a first step (always start by the lowest library in the dependency tree of the project).

Let's generate the documenation for Gdk with meson. In that example, we will add the meson snipnet in docs/reference/gdk/meson.build.

To simplify the transcription of that documentation to other projects, we will export the HOTDOC_DOCUMENTATION_ROOT variable like that:

# REPLACE THE PATH WITH YOUR OWN PATH.
export HOTDOC_DOCUMENTATION_ROOT=$PWD/docs/references/gdk

The following files will be required first:

  • docs/reference/gdk/sitemap.txt (run echo 'gi-index' > $HOTDOC_DOCUMENTATION_ROOT/sitemap.txt)
  • docs/reference/gdk/markdown/index.md (run mkdir $HOTDOC_DOCUMENTATION_ROOT/markdown && touch $HOTDOC_DOCUMENTATION_ROOT/markdown/index.md)

hotdoc = import('hotdoc')

hotdoc.generate_doc('Gdk',
  index: 'markdown/index.md',
  project_version: gtk_api_version,
  sitemap: 'sitemap.txt',
  gi_smart_index: true,
  gi_sources: gdk_gir[0].full_path(),
  gi_c_sources: gdk_public_headers + gdk_public_sources,
  gi_c_source_roots: src_dir,
  dependencies: [gdk_gir],
)
